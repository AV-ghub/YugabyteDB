## Сводная матрица инструментов ↔ систем (теоретическая часть)

| Инструмент | Направление | YugabyteDB (источник/приёмник) | Целевая система | Поддержка обратного пути (целевая → Yugabyte) | Краткое описание пайплайна |
|------------|-------------|--------------------------------|----------------|----------------------------------------------|----------------------------|
| **FDW** (Foreign Data Wrapper) | **Вставка в другую СУБД** (Yugabyte → target) | `CREATE EXTENSION yugabyte_fdw` (или `postgres_fdw` в YSQL) | PostgreSQL, SQLite (через `sqlite_fdw`), ClickHouse (через `clickhouse_fdw`), Elasticsearch (через `elasticsearch_fdw`) | **Да**, если целевая СУБД тоже поддерживает FDW к Yugabyte (например, `postgres_fdw` в PostgreSQL) | Yugabyte‑сервер выступает как foreign server; запрос `INSERT … SELECT` проходит через FDW, данные копируются без промежуточных файлов. |
| **FDW** | **Выборка из другой СУБД** (target → Yugabyte) | `CREATE SERVER foreign_pg FOREIGN DATA WRAPPER postgres_fdw OPTIONS (host 'pg', dbname 'test');` | PostgreSQL, SQLite, ClickHouse, Elasticsearch | **Да**, если у Yugabyte есть FDW‑клиент к целевой СУБД (обычно реализуется в YSQL) | Yugabyte читает внешние таблицы как обычные локальные, позволяя выполнять `SELECT … FROM foreign_table`. |
| **COPY / \COPY** | **Вставка (batch) в другую СУБД** | `COPY (SELECT …) TO PROGRAM 'curl -XPOST http://clickhouse:8123/…' CSV` | Любая система, принимающая CSV/JSON/Parquet (ClickHouse, DuckDB, SQLite, Elasticsearch via Bulk API) | **Да**, если целевая система умеет принимать `COPY FROM STDIN` (PostgreSQL, SQLite) | Одноразовый экспорт‑импорт; подходит для миграций и периодических выгрузок. |
| **COPY FROM** | **Выборка (batch) из другой СУБД** | `COPY target_table FROM '/data/file.csv' CSV` | Любая система, способная экспортировать CSV/Parquet (Yugabyte, PostgreSQL, ClickHouse, DuckDB) | **Да**, если Yugabyte умеет `COPY FROM` (поддерживается в YSQL) | Обратный процесс: выгрузка из внешней СУБД → файл → `COPY FROM` в Yugabyte. |
| **Файловый обмен** (CSV, JSON, Parquet, Avro) | **Вставка** | `EXPORT TO '/tmp/orders.parquet'` (Yugabyte → Parquet) | DuckDB, ClickHouse, SQLite, Elasticsearch (Bulk JSON), Kafka (Avro) | **Да**, если целевая система может экспортировать в тот же формат | Пайплайн: Yugabyte `COPY TO` → файл → `IMPORT`/`INSERT` в целевую СУБД. |
| **Файловый обмен** | **Выборка** | `IMPORT FROM '/tmp/orders.parquet'` | Любая система, умеющая импортировать тот же формат | **Да**, если Yugabyte может импортировать (CSV/Parquet) | Обратный процесс аналогичен выше. |
| **CDC + Kafka** | **Вставка (push‑only)** | Встроенный Yugabyte CDC‑producer → Kafka‑topic | ClickHouse (Kafka‑engine), Elasticsearch (Kafka‑sink), Redis (Kafka‑→ Redis‑bridge), PostgreSQL (Kafka‑sink), DuckDB (Kafka‑consumer → `INSERT`) | **Да**, если целевая система тоже генерирует CDC в Kafka (PostgreSQL logical decoding, Kafka Connect source) | Поток изменений в реальном времени; минимум задержки, без файлов. |
| **CDC + Debezium** | **Вставка** | Debezium source‑connector читает лог Yugabyte → Kafka → sink‑connector в целевую СУБД | PostgreSQL, ClickHouse, Elasticsearch, Redis, DuckDB (через JDBC‑sink) | **Да**, Debezium source‑connector может читать лог PostgreSQL → Kafka → Yugabyte CDC‑consumer | Позволяет использовать единый Kafka‑cluster для всех источников/приёмников. |
| **CDC + RabbitMQ** | **Вставка** | Yugabyte CDC → RabbitMQ‑producer (через `kafka‑rabbitmq‑bridge` или custom) | RabbitMQ‑queues, далее consumer‑приложения | **Да**, RabbitMQ → Yugabyte через consumer‑producer → Kafka → Yugabyte CDC‑consumer | Подходит, если инфраструктура уже использует RabbitMQ вместо Kafka. |
| **JDBC/ODBC** | **Вставка** | `INSERT …` через JDBC‑драйвер Yugabyte в целевую СУБД | PostgreSQL, SQLite, ClickHouse (JDBC), DuckDB (JDBC), Elasticsearch (JDBC‑like HTTP) | **Да**, если целевая СУБД предоставляет JDBC/ODBC‑драйвер к Yugabyte (PostgreSQL‑compatible) | Прямой клиент‑сервер, подходит для небольших объёмов и ad‑hoc запросов. |
| **SQL‑based API** (REST, HTTP) | **Вставка** | Yugabyte → ClickHouse HTTP‑INSERT, Elasticsearch Bulk API | ClickHouse, Elasticsearch | **Да**, если целевая система предоставляет API для чтения (Elasticsearch → Yugabyte: `_search` → ETL) | Позволяет обходить драйверы, использовать обычные HTTP‑клиенты. |
| **Message‑queue bridge** (Kafka ↔ RabbitMQ) | **Двусторонний** | CDC → Kafka ↔ RabbitMQ ↔ Yugabyte | Любая из систем, подключённая к одному из брокеров | **Да**, если обе стороны поддерживают один из брокеров | Универсальный слой, упрощает добавление новых систем без изменения CDC‑логики. |
| **Custom ETL‑скрипт** (Python/Go) | **Вставка/Выборка** | `psycopg2`/`pyyugabyte` → `redis-py`/`elasticsearch-py` | Любая система | **Да**, если реализовать обратный путь в том же скрипте | Гибкий, но требует разработки и поддержки. |
