### **FDW (Foreign Data Wrapper)**

**Применимость:** Запросы к внешним таблицам, JOIN с локальными данными, перенос данных через `INSERT ... SELECT`

**Описание:**
YugabyteDB выступает как foreign server; данные читаются напрямую без промежуточных файлов. Работает через стандартный `postgres_fdw` (входит в поставку PostgreSQL). Для некоторых систем требуются дополнительные FDW-расширения.

**Пример использования на стороне YugabyteDB:**
```sql
-- Подключение к внешнему PostgreSQL
CREATE EXTENSION postgres_fdw;
CREATE SERVER foreign_pg FOREIGN DATA WRAPPER postgres_fdw 
  OPTIONS (host 'pg-host', port '5432', dbname 'source');
CREATE USER MAPPING FOR CURRENT_USER SERVER foreign_pg 
  OPTIONS (user 'user', password 'pass');
CREATE FOREIGN TABLE pg_users (id INT, name TEXT) 
  SERVER foreign_pg OPTIONS (table_name 'users');

-- Теперь можно делать SELECT и INSERT...SELECT
INSERT INTO local_table SELECT * FROM pg_users;
```

| Внешняя система | Вставка (YB → Система) | Выборка (YB ← Система) | Возможность обратного взаимодействия | Краткие замечания | Ссылки на первоисточник |
|-----------------|:----------------------:|:----------------------:|:------------------------------------:|-------------------|------------------------|
| **PostgreSQL** | ✅ `postgres_fdw` | ✅ `postgres_fdw` | ✅ Полная | Наиболее стабильное решение. Работает в обе стороны. | [Postgres FDW](https://www.postgresql.org/docs/current/postgres-fdw.html) |
| **ClickHouse** | ❌ | ✅ `clickhouse_fdw` | ❌ | Только чтение. Требует установки расширения. Не поддерживает запись. | [ClickHouse FDW](https://github.com/ildus/clickhouse_fdw) |
| **DuckDB** | ❌ | ⚠️ `postgres_scanner` | ❌ | DuckDB может читать из PostgreSQL (и YB), но не наоборот. FDW для DuckDB в YB не существует. | [DuckDB Postgres Scanner](https://duckdb.org/docs/extensions/postgres_scanner) |
| **SQLite** | ❌ | ❌ | ❌ | SQLite не является сервером, к нему нельзя подключиться по сети. | - |
| **Redis** | ❌ | ❌ | ❌ | Нет FDW, другая модель данных. | - |
| **Elasticsearch** | ❌ | ✅ `elasticsearch_fdw` | ❌ | Только чтение. Требует установки расширения. | [Elasticsearch FDW](https://github.com/enterprisedb/elasticsearch_fdw) |
| **Kafka** | ❌ | ❌ | ❌ | Нет FDW, есть коннекторы Kafka Connect. | - |
| **RabbitMQ** | ❌ | ❌ | ❌ | Нет FDW. | - |

### **COPY (команда PostgreSQL/YSQL)**

**Применимость:** Потоковый импорт/экспорт данных, миграции, бэкапы

**Описание:**
Нативный протокол PostgreSQL для высокоскоростного обмена данными в текстовом/бинарном формате. Работает через stdin/stdout или файлы.

**Пример использования:**
```bash
# Экспорт из YugabyteDB в CSV
psql -h yb-host -d db -c "\copy table TO 'export.csv' CSV HEADER"

# Импорт в YugabyteDB из CSV
psql -h yb-host -d db -c "\copy table FROM 'import.csv' CSV HEADER"
```

| Внешняя система | Вставка (YB → Система) | Выборка (YB ← Система) | Возможность обратного взаимодействия | Краткие замечания | Ссылки на первоисточник |
|-----------------|:----------------------:|:----------------------:|:------------------------------------:|-------------------|------------------------|
| **PostgreSQL** | ✅ `COPY TO` | ✅ `COPY FROM` | ✅ Полная | Полная совместимость форматов. Можно напрямую копировать между БД через пайпы. | [PostgreSQL COPY](https://www.postgresql.org/docs/current/sql-copy.html) |
| **ClickHouse** | ⚠️ `COPY TO` → CSV → `clickhouse-client` | ⚠️ `clickhouse-client` → CSV → `COPY FROM` | ❌ | ClickHouse поддерживает импорт/экспорт CSV, но не протокол COPY. Требуется промежуточный файл. | [ClickHouse Formats](https://clickhouse.com/docs/en/interfaces/formats) |
| **DuckDB** | ⚠️ `COPY TO` → CSV → `COPY` в DuckDB | ⚠️ DuckDB → CSV → `COPY FROM` в YB | ❌ | DuckDB имеет команду `COPY`, но формат не полностью совместим. | [DuckDB Import](https://duckdb.org/docs/data/overview) |
| **SQLite** | ⚠️ `COPY TO` → CSV → `.import` | ⚠️ `.dump`/`.mode csv` → `psql` | ❌ | SQLite не поддерживает COPY. Только через CSV. | [SQLite Import](https://www.sqlite.org/cli.html#csv_import) |
| **Redis** | ❌ | ❌ | ❌ | Неприменимо. | - |
| **Elasticsearch** | ❌ | ❌ | ❌ | Неприменимо. | - |
| **Kafka** | ❌ | ❌ | ❌ | Неприменимо. | - |
| **RabbitMQ** | ❌ | ❌ | ❌ | Неприменимо. | - |

### **Файловый обмен (CSV, JSON, Parquet, AVRO)**

**Применимость:** Асинхронный обмен, ETL-процессы, архивация, обмен с системами без прямого сетевого доступа

**Описание:** Универсальный способ через файлы-посредники. Подходит для любых систем, но требует ручного или скриптового управления.

| Внешняя система | Экспорт (YB → Файл → Система) | Импорт (Система → Файл → YB) | Краткие замечания | Ссылки на первоисточник |
|-----------------|:-----------------------------:|:----------------------------:|-------------------|------------------------|
| **PostgreSQL** | ✅ CSV/JSON через `COPY` | ✅ CSV/JSON через `COPY` | Нативные инструменты с обеих сторон | [PostgreSQL COPY](https://www.postgresql.org/docs/current/sql-copy.html) |
| **ClickHouse** | ✅ CSV/Parquet/JSON через `clickhouse-client` | ✅ CSV/Parquet/JSON через `COPY` | ClickHouse отлично читает внешние форматы | [ClickHouse Formats](https://clickhouse.com/docs/en/interfaces/formats) |
| **DuckDB** | ✅ CSV/Parquet/JSON через `COPY` или `EXPORT` | ✅ CSV/Parquet/JSON через `COPY` | DuckDB специализируется на чтении файлов | [DuckDB Data Import](https://duckdb.org/docs/data/overview) |
| **SQLite** | ✅ CSV/JSON через `.mode csv` + `.output` | ✅ CSV/JSON через `.import` | Требуется конвертация | [SQLite CLI](https://www.sqlite.org/cli.html) |
| **Redis** | ⚠️ JSON через скрипты (`redis-cli --json`) | ⚠️ JSON → Redis через скрипты | Только через промежуточную обработку | [Redis Mass Insert](https://redis.io/docs/manual/mass-insert/) |
| **Elasticsearch** | ⚠️ JSON через bulk API | ⚠️ JSON в YB через скрипты | ES умеет экспорт в JSON | [Elasticsearch Export](https://www.elastic.co/guide/en/elasticsearch/reference/current/docs.html) |
| **Kafka** | ❌ | ❌ | Kafka — стриминговая платформа, не хранит данные постоянно | - |
| **RabbitMQ** | ❌ | ❌ | Очереди сообщений, не хранят данные постоянно | - |

### **CDC (Change Data Capture) + Kafka**

**Применимость:** Стриминг изменений в реальном времени, построение data lakes, репликация, аудит

**Описание:** YugabyteDB поддерживает CDC через Debezium-совместимый коннектор. Изменения попадают в Kafka, откуда могут быть прочитаны другими системами.

**Архитектура:**
```
YugabyteDB → Debezium → Kafka → Sink-коннектор → Целевая система
```

| Внешняя система | YB → Система (CDC) | Система → YB (CDC) | Возможность обратного взаимодействия | Краткие замечания | Ссылки на первоисточник |
|-----------------|:------------------:|:------------------:|:------------------------------------:|-------------------|------------------------|
| **PostgreSQL** | ✅ Debezium (YB → Kafka) | ✅ Debezium (PG → Kafka) | ✅ Полная | Обе системы поддерживают Debezium | [Debezium](https://debezium.io/documentation/) |
| **ClickHouse** | ✅ YB CDC → Kafka → CH Kafka Engine | ❌ | ❌ | CH только потребляет через Kafka Engine | [ClickHouse Kafka](https://clickhouse.com/docs/en/engines/table-engines/integrations/kafka) |
| **DuckDB** | ❌ | ❌ | ❌ | Не поддерживает CDC | - |
| **SQLite** | ❌ | ❌ | ❌ | Не поддерживает CDC | - |
| **Redis** | ❌ | ❌ | ❌ | Не поддерживает CDC | - |
| **Elasticsearch** | ✅ YB CDC → Kafka → ES sink | ❌ | ❌ | Только потребление через Logstash или Kafka Connect | [ES Kafka](https://www.elastic.co/guide/en/elasticsearch/reference/current/debezium-connector.html) |
| **Kafka** | ✅ YB CDC → Kafka | ✅ Kafka → YB (sink) | ✅ Полная | Kafka как транспорт | [YB CDC Docs](https://docs.yugabyte.com/preview/integrations/cdc/debezium/) |
| **RabbitMQ** | ⚠️ YB CDC → приложение → RMQ | ⚠️ RMQ → приложение → YB | ⚠️ Частичная | Нет готовых коннекторов, только через код | - |

### **Kafka Connect**

**Применимость:** Управляемый стриминг данных между Kafka и внешними системами

**Описание:** Готовая платформа для подключения Kafka к различным системам. Source коннекторы читают из систем в Kafka, Sink коннекторы пишут из Kafka в системы.

| Система | Source (из системы в Kafka) | Sink (из Kafka в систему) | Направление | Краткие замечания | Ссылки |
|---------|:---------------------------:|:-------------------------:|:-----------:|-------------------|--------|
| **PostgreSQL** | ✅ Debezium PG source | ✅ JDBC sink | ↔ | Зрелое решение | [Kafka Connect JDBC](https://docs.confluent.io/kafka-connect-jdbc/current/) |
| **YugabyteDB** | ✅ Debezium YB source | ✅ JDBC sink | ↔ | Аналогично PostgreSQL | [YB Kafka](https://docs.yugabyte.com/preview/integrations/cdc/debezium/) |
| **ClickHouse** | ❌ | ✅ ClickHouse sink | → | Только запись | [ClickHouse Kafka](https://clickhouse.com/docs/en/integrations/kafka) |
| **Elasticsearch** | ❌ | ✅ Elasticsearch sink | → | Только запись | [ES Kafka Connect](https://www.elastic.co/guide/en/elasticsearch/reference/current/debezium-connector.html) |
| **Redis** | ❌ | ✅ Redis sink | → | Только запись | [Redis Sink](https://docs.confluent.io/kafka-connect-redis/current/) |
| **SQLite** | ❌ | ❌ | - | Не поддерживается | - |
| **DuckDB** | ❌ | ❌ | - | Не поддерживается | - |
| **RabbitMQ** | ⚠️ | ⚠️ | ⚠️ | Есть экспериментальные коннекторы | [RMQ Kafka](https://github.com/rabbitmq/rabbitmq-kafka) |

### **Прямое клиентское подключение (Native drivers)**

**Применимость:** Приложение выступает оркестратором, читает из одной системы и пишет в другую

**Описание:** Самый гибкий, но и самый трудозатратный способ. Приложение на Python/Java/Go одновременно подключено к обоим системам и управляет передачей данных.

| Внешняя система | YB → Система (приложение читает из YB и пишет в систему) | Система → YB (приложение читает из системы и пишет в YB) | Краткие замечания | Ссылки |

| **ClickHouse** | ✅ CH клиент + YB клиент | ✅ CH клиент + YB клиент | Разные драйверы, но приложение управляет логикой | [CH Drivers](https://clickhouse.com/docs/en/integrations/language-clients) |

### **Выводы и рекомендации**

1. **PostgreSQL** — единственная система с полной двусторонней поддержкой всех инструментов. Наиболее простой путь интеграции.

2. **Kafka** — идеальный "транспортный хаб" для CDC и потоковой интеграции. Рекомендуется как основа для построения пайплайнов в реальном времени.

3. **ClickHouse и Elasticsearch** — только потребители данных. Оптимальный способ — CDC через Kafka.

4. **SQLite и DuckDB** — требуют ручной синхронизации через файлы или приложение. Подходят для периодического обмена.

5. **Redis** — только через приложение или Kafka Connect (sink). YEDIS API устарел и не рекомендуется.

6. **RabbitMQ** — требует написания кода-моста. Нет готовых коннекторов.

**Рекомендуемый стек для новых интеграций:**
- Для real-time: YugabyteDB → Kafka → целевая система
- Для batch: YugabyteDB → файлы (Parquet/CSV) → целевая система
- Для запросов: YugabyteDB ↔ PostgreSQL через FDW
